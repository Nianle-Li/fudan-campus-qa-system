# 系统核心功能开发与阶段整合演示说明

本文档用于说明当前阶段前端、后端与数据库的联动成果，以及课堂/验收演示时可使用的操作路径。

## 一、阶段实现范围

本阶段在已有数据库设计和初始化脚本基础上，补齐了一个可运行的系统纵向切片：

- 数据库：使用 PostgreSQL，建表、约束、索引和初始测试数据由 `db/init.sql` 统一导入。
- 后端：`backend/app.py` 提供 HTTP API，将前端请求转换为 PostgreSQL 查询和写入操作。
- 前端：`frontend/index.html` 提供数据查询、基础维护和关键 SQL 展示页面。
- 演示文档：本说明文档描述启动步骤、核心业务场景和关键 SQL 效果。

## 二、运行准备

### 1. 初始化数据库

在仓库根目录执行：

```bash
createdb fcqa
psql -v ON_ERROR_STOP=1 -d fcqa -f db/init.sql
```

如只需要建表和索引，不导入测试数据，可执行：

```bash
psql -v ON_ERROR_STOP=1 -d fcqa -f db/schema.sql
```

### 2. 安装后端依赖

后端 HTTP 服务使用 Python 标准库，连接 PostgreSQL 需要 `psycopg`：

```bash
python -m pip install -r backend/requirements.txt
```

### 3. 启动后端并打开前端

PowerShell 示例：

```powershell
$env:DATABASE_URL="postgresql://postgres:postgres@localhost:5432/fcqa"
python backend/app.py
```

浏览器访问：

```text
http://127.0.0.1:8000
```

如果当前机器没有 PostgreSQL，可临时使用演示模式预览前后端交互：

```powershell
$env:FCQA_DEMO_MODE="1"
python backend/app.py
```

演示模式只使用内置样例数据，正式验收应连接 PostgreSQL。

## 三、核心功能演示路径

### 0. 自然语言查询

页面顶部“自然语言查询”区域支持输入中文问句，后端会识别查询意图、生成 SQL、执行查询并返回结果。可演示的问题包括：

- `邯郸校区有哪些教学楼？`
- `第二教学楼有哪些自习室？`
- `李芳老师教什么课？`
- `数据库系统原理周几上课？`
- `近期有哪些校园活动？`
- `热门活动有哪些？`

每次查询都会展示识别意图、查询类别、SQL 参数、SQL 文本和结果表格。正式 PostgreSQL 模式下，原始问句会写入 `query_log`，用于热门查询类别统计。

### 1. 数据查询

打开首页后，在左侧“数据查询”区域选择对象并输入关键词：

- 校区：查询邯郸、江湾、枫林、张江等校区基础信息。
- 建筑：查询建筑名称、类型及所属校区。
- 设施：查询设施名称、类型、开放时间及所属建筑。
- 课程：展示课程、课程代码、授课教师和排课地点。
- 活动：展示活动名称、时间、主办单位、举办设施和参与人数。

### 2. 数据新增

在中间维护区域可新增三类核心数据：

- 建筑：填写建筑名称、类型、所属校区后保存。
- 设施：填写设施名称、类型、开放时间、所属建筑后保存。
- 活动：填写活动名称、主办单位、时间、设施和简介后保存。

保存成功后左侧列表会自动刷新，可立即查询新增结果。

### 3. 数据修改

在左侧列表选择“建筑 / 设施 / 活动”后点击“编辑”，记录会自动载入中间表单。修改字段后保存，即可调用后端 `PUT` 接口更新数据库。

### 4. 数据删除

编辑某条建筑、设施或活动后点击“删除”。

数据库外键约束会限制不安全删除，例如仍被设施引用的建筑、仍被活动或课程时段引用的设施，在 PostgreSQL 中会被 `ON DELETE RESTRICT` 拒绝，便于展示约束生效效果。

### 5. 查询记录写入

切换到“查询记录”表单，填写用户 ID、查询类别和查询内容后保存。该操作会向 `query_log` 表插入记录，可用于后续热门查询类别统计。

## 四、关键 SQL 查询效果

右侧“关键 SQL 效果”区域会调用：

```text
GET /api/sql-examples
```

后端会实际执行以下 SQL 并返回结果：

1. 查询邯郸校区建筑列表：验证 `campus -> building` 关联。
2. 查询第二教学楼设施：验证 `building -> facility` 关联。
3. 查询李芳老师授课列表：验证 `users/teacher/course_offering/course` 多表 JOIN。
4. 查询近期校园活动：验证 `activity -> facility -> building -> campus` 关联。
5. 统计每个校区各类建筑数量：验证分组统计。
6. 统计热门查询类别：验证 `query_log` 聚合分析。
7. 统计热门活动：验证 `activity/user_activity` 关联统计。

同样的 SQL 也可通过命令行执行：

```bash
psql -v ON_ERROR_STOP=1 -d fcqa -f db/verify_basic_queries.sql
```

## 五、接口与数据库表对应

| 功能 | API | 主要数据表 |
| :--- | :--- | :--- |
| 健康检查 | `GET /api/health` | 数据库连接 |
| 校区查询 | `GET /api/campuses` | `campus` |
| 建筑维护 | `GET/POST/PUT/DELETE /api/buildings` | `building`, `campus` |
| 设施维护 | `GET/POST/PUT/DELETE /api/facilities` | `facility`, `building` |
| 课程查询 | `GET /api/courses` | `course`, `course_section`, `course_offering`, `course_offering_teacher`, `course_offering_schedule` |
| 活动维护 | `GET/POST/PUT/DELETE /api/activities` | `activity`, `facility` |
| 查询记录写入 | `POST /api/query-log` | `query_log` |
| 自然语言查询 | `POST /api/nl-query` | 多表查询，查询记录写入 `query_log` |
| SQL 展示 | `GET /api/sql-examples` | 多表 JOIN 与聚合查询 |
