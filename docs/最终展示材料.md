# 最终展示材料

本文档可作为最后一次课项目演讲的讲稿提纲和现场演示脚本。

## 一、项目标题

复旦校园百事通问答系统：面向校园信息查询的关系数据库设计与联动演示

## 二、展示目标

本项目构建一个以 PostgreSQL 为核心的数据管理与查询系统，统一存储校区、建筑、设施、课程、教师、活动和用户行为记录，并通过前端、后端、数据库联动展示基础问答场景所需的数据新增、查询、修改、删除和统计分析能力。

## 三、系统架构

1. 前端：静态 HTML/CSS/JavaScript 工作台。
2. 后端：Python HTTP API，负责参数校验、SQL 执行和 JSON 返回。
3. 数据库：PostgreSQL，负责实体存储、约束校验、索引优化和统计查询。

数据流：

```text
浏览器界面 -> 后端 API -> PostgreSQL -> 后端 JSON -> 浏览器表格/SQL 展示
```

## 四、数据库设计亮点

- 采用 14 张核心表覆盖用户、教师、学生、校区、建筑、设施、课程、课程班次、课程开设、课程时段、授课教师、活动、查询记录和用户活动参与。
- 教师/学生使用共享主键建模，避免公共属性重复。
- 课程拆分为课程本体、课程班次、课程开设、排课时段和授课教师，避免教师、地点、学期混入课程定义层。
- 活动通过设施关联到建筑和校区，支持活动地点追溯。
- 使用主键、外键、唯一、非空、默认值、`CHECK` 约束维护实体完整性、参照完整性和域完整性。
- 使用 B-tree、GIN、trigram 和生成列支持关联查询、分类统计与模糊检索。

## 五、系统功能展示顺序

### 1. 首页与健康检查

打开：

```text
http://127.0.0.1:8000
```

讲解右上角连接状态：正常情况下显示后端与数据库模式；数据库不可达时会提示未连接。

### 2. 自然语言查询

在顶部问句框中依次演示：

- `邯郸校区有哪些教学楼？`
- `第二教学楼有哪些自习室？`
- `李芳老师教什么课？`
- `数据库系统原理周几上课？`
- `近期有哪些校园活动？`
- `热门活动有哪些？`

讲解系统会将自然语言问句映射为数据库查询意图，并展示实际 SQL、参数和结果表格。正式 PostgreSQL 模式下，问句会写入 `query_log`。

### 3. 数据查询

在左侧依次选择：

- 校区：展示校园空间基础数据。
- 建筑：展示建筑类型和校区归属。
- 设施：展示开放时间和建筑归属。
- 课程：展示课程代码、授课教师和排课安排。
- 活动：展示活动时间、主办单位、设施和参与人数。

### 4. 数据维护

演示新增、修改、删除：

1. 新增建筑：“最终演示楼”，类型选“教学楼”，校区选“邯郸校区”。
2. 点击左侧建筑列表中的“编辑”，将类型改为“综合楼”并保存。
3. 新增设施：“最终演示教室”，绑定到刚才的建筑。
4. 新增活动：“最终项目展示彩排”，绑定到该设施。
5. 展示删除顺序：先删活动，再删设施，最后删建筑，说明外键约束对删除顺序的保护。

### 5. 查询记录写入

在“查询记录”表单中写入：

```text
用户 ID：4
查询类别：活动
查询内容：近期有哪些校园活动
```

说明该记录进入 `query_log`，可用于热门查询类别统计。

### 6. 关键 SQL 效果

点击右侧“执行”，展示以下 SQL 的文本和实际结果：

- 查询某校区建筑。
- 查询某建筑设施。
- 查询某教师授课列表。
- 查询近期活动及其校区/建筑/设施。
- 统计每个校区各类建筑数量。
- 统计热门查询类别。
- 统计热门活动。

## 六、可现场运行的命令

初始化数据库：

```bash
createdb fcqa
psql -v ON_ERROR_STOP=1 -d fcqa -f db/init.sql
```

启动正式服务：

```powershell
$env:DATABASE_URL="postgresql://postgres:postgres@localhost:5432/fcqa"
python backend/app.py
```

无数据库时启动演示服务：

```powershell
$env:FCQA_DEMO_MODE="1"
python backend/app.py
```

运行自动自测：

```bash
python scripts/smoke_test.py --start-demo
```

执行 SQL 验收：

```bash
psql -v ON_ERROR_STOP=1 -d fcqa -f db/verify_basic_queries.sql
```

## 七、答辩要点

- 数据库不是单纯堆表，而是把稳定信息、实例信息和时段信息分层建模。
- 约束在数据库层实现，非法数据会被数据库拒绝，减少应用层遗漏校验导致的不一致。
- 前后端联动覆盖了自然语言查询、核心 CRUD 和关键 SQL 展示，能够支撑后续问答增强或后台管理扩展。
- 演示模式保证课堂展示稳定；PostgreSQL 模式保证最终实现与真实数据库设计一致。
