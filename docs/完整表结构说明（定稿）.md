# 完整表结构说明（定稿）

本文档在《需求分析说明书》和《关系模式初稿及说明》的基础上，对复旦校园百事通系统的数据库表结构进行定稿说明。每张表均明确字段类型、主键、外键、非空、唯一、检查约束和默认值，便于后续数据库物理实现、数据导入和约束验收。

## 一、统一约束约定

1. 主键字段默认同时具有 `NOT NULL` 和 `UNIQUE` 语义。
2. 自增编号字段使用 `INT AUTO_INCREMENT`，用于实体表的代理主键。
3. 外键默认采用 `ON UPDATE CASCADE`；删除策略按业务语义分别设置为 `RESTRICT` 或 `CASCADE`。
4. 字符串字段如标注“非空字符串”，表示建议增加 `CHECK (TRIM(字段名) <> '')`。
5. 枚举类字段采用 `CHECK (字段名 IN (...))` 约束；若具体数据库版本不支持 `CHECK`，应在应用层或触发器中保证同等规则。

## 二、用户表 User

用户表保存系统中所有用户共有的基础信息，是教师表、学生表和查询记录表等表的上位引用表。

| 字段名 | 字段描述 | 数据类型 | 键约束 | 非空 | 唯一 | 检查约束或默认值 | 说明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| `user_id` | 用户ID | `INT AUTO_INCREMENT` | 主键 | 是 | 是 | 默认自增 | 唯一标识一个用户 |
| `name` | 姓名 | `VARCHAR(50)` | 无 | 是 | 否 | `CHECK (TRIM(name) <> '')` | 用户真实姓名或展示姓名 |
| `department` | 院系 | `VARCHAR(100)` | 无 | 是 | 否 | `DEFAULT '未填写'`，`CHECK (TRIM(department) <> '')` | 院系允许暂缺，但用默认值避免空值 |

表级约束：

- `PRIMARY KEY (user_id)`

## 九、课程班次表 CourseSection

课程班次表记录“课程代码”与“课程主代码”的对应关系。课程代码用于表示可重复使用的课程班次模板。

| 字段名 | 字段描述 | 数据类型 | 键约束 | 非空 | 唯一 | 检查约束或默认值 | 说明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| `course_code` | 课程代码 | `VARCHAR(30)` | 主键 | 是 | 是 | `CHECK (TRIM(course_code) <> '')`，`CHECK (course_code LIKE CONCAT(course_master_code, '.%'))` | 如 `COMP130015.01`，应以对应课程主代码加英文句点开头 |
| `course_master_code` | 课程主代码 | `VARCHAR(20)` | 外键 | 是 | 否 | 无默认值 | 关联 `Course(course_master_code)` |

表级约束：

- `PRIMARY KEY (course_code)`
- `CHECK (course_code LIKE CONCAT(course_master_code, '.%'))`
- `FOREIGN KEY (course_master_code) REFERENCES Course(course_master_code) ON UPDATE CASCADE ON DELETE RESTRICT`

## 十、课程开设表 CourseOffering

课程开设表记录某个课程代码在某一学期的一次实际开设。由于同一课程代码在同一学期可能并行开设，因此使用独立的 `offering_id` 区分实例。

| 字段名 | 字段描述 | 数据类型 | 键约束 | 非空 | 唯一 | 检查约束或默认值 | 说明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| `offering_id` | 开设编号 | `INT AUTO_INCREMENT` | 主键 | 是 | 是 | 默认自增 | 唯一标识一次实际开设 |
| `course_code` | 课程代码 | `VARCHAR(30)` | 外键 | 是 | 否 | 无默认值 | 关联 `CourseSection(course_code)` |
| `semester` | 开课学期 | `VARCHAR(50)` | 无 | 是 | 否 | `CHECK (TRIM(semester) <> '')` | 如 `2024-2025 秋季学期` |

表级约束：

- `PRIMARY KEY (offering_id)`
- `FOREIGN KEY (course_code) REFERENCES CourseSection(course_code) ON UPDATE CASCADE ON DELETE RESTRICT`
- 不设置 `UNIQUE (course_code, semester)`，以保留同一课程代码在同一学期并行开设的可能性。

## 十一、课程开设时段表 CourseOfferingSchedule

课程开设时段表记录每次课程开设的具体上课时间和地点。一个课程开设可对应多个时段，不同时段可使用不同设施。

| 字段名 | 字段描述 | 数据类型 | 键约束 | 非空 | 唯一 | 检查约束或默认值 | 说明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| `offering_id` | 开设编号 | `INT` | 复合主键，外键 | 是 | 复合唯一 | 无默认值 | 关联 `CourseOffering(offering_id)` |
| `day_of_week` | 周内上课日 | `VARCHAR(10)` | 复合主键 | 是 | 复合唯一 | `CHECK (day_of_week IN ('周一', '周二', '周三', '周四', '周五', '周六', '周日'))` | 标识一周中的上课日 |
| `start_period` | 开始节次 | `INT` | 复合主键 | 是 | 复合唯一 | `CHECK (start_period BETWEEN 1 AND 13)` | 起始节次 |
| `end_period` | 结束节次 | `INT` | 复合主键 | 是 | 复合唯一 | `CHECK (end_period BETWEEN 1 AND 13)` | 结束节次 |
| `week_type` | 周次类型 | `VARCHAR(20)` | 复合主键 | 是 | 复合唯一 | `DEFAULT '每周'`，`CHECK (week_type IN ('每周', '单周', '双周'))` | 用于表示单双周安排 |
| `facility_id` | 设施ID | `INT` | 外键 | 是 | 否 | 无默认值 | 关联 `Facility(facility_id)` |

表级约束：

- `PRIMARY KEY (offering_id, day_of_week, start_period, end_period, week_type)`
- `CHECK (start_period <= end_period)`
- `FOREIGN KEY (offering_id) REFERENCES CourseOffering(offering_id) ON UPDATE CASCADE ON DELETE CASCADE`
- `FOREIGN KEY (facility_id) REFERENCES Facility(facility_id) ON UPDATE CASCADE ON DELETE RESTRICT`

## 十三、活动表 Activity

活动表保存校园活动的基础信息。活动必须绑定到一个具体设施，从而可以进一步追溯到建筑和校区。

| 字段名 | 字段描述 | 数据类型 | 键约束 | 非空 | 唯一 | 检查约束或默认值 | 说明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| `activity_id` | 活动ID | `INT AUTO_INCREMENT` | 主键 | 是 | 是 | 默认自增 | 唯一标识一个活动 |
| `name` | 活动名称 | `VARCHAR(150)` | 无 | 是 | 否 | `CHECK (TRIM(name) <> '')` | 活动标题 |
| `description` | 活动简介 | `TEXT` | 无 | 是 | 否 | `DEFAULT '暂无简介'` | 用于详情展示、关键词检索和问答返回 |
| `start_time` | 开始时间 | `DATETIME` | 无 | 是 | 否 | 无默认值 | 活动开始时间 |
| `end_time` | 结束时间 | `DATETIME` | 无 | 否 | 否 | `DEFAULT NULL` | 可为空，表示暂未确定或瞬时活动 |
| `organizer` | 主办单位 | `VARCHAR(100)` | 无 | 是 | 否 | `DEFAULT '未填写'`，`CHECK (TRIM(organizer) <> '')` | 主办院系、社团或部门 |
| `facility_id` | 举办设施ID | `INT` | 外键 | 是 | 否 | 无默认值 | 关联 `Facility(facility_id)` |

表级约束：

- `PRIMARY KEY (activity_id)`
- `UNIQUE (name, start_time, facility_id)`
- `CHECK (end_time IS NULL OR end_time >= start_time)`
- `FOREIGN KEY (facility_id) REFERENCES Facility(facility_id) ON UPDATE CASCADE ON DELETE RESTRICT`

说明：

- `description` 使用默认值是为了避免活动简介为空时影响活动展示。若具体数据库不允许 `TEXT` 字段设置默认值，可由应用层在插入时写入 `'暂无简介'`。
