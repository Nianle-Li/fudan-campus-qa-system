# 自我测试说明

本文档记录最终提交前的自测范围、测试方法、测试结果和剩余风险，用于支撑最后一次课的项目展示与同伴评审。

## 一、测试环境

- 操作系统：Windows，本地 PowerShell
- 后端运行环境：Python 3.13
- 前端运行方式：后端静态托管 `frontend/`
- 数据库目标环境：PostgreSQL 16 兼容版本
- 本地演示验证：`FCQA_DEMO_MODE=1`
- PostgreSQL 驱动：`psycopg[binary]`

说明：本机当前没有可连接的 PostgreSQL 服务，因此本地自动化测试以演示数据模式验证前后端稳定性；正式数据库联调可按 `db/README.md` 初始化 PostgreSQL 后运行同一套后端 API 和 SQL 验收脚本。

## 三、数据库一致性测试

数据库脚本层面提供两类验证：

### 1. 初始化脚本

```bash
createdb fcqa
psql -v ON_ERROR_STOP=1 -d fcqa -f db/init.sql
```

验证目标：

- 扩展、表结构和索引能按顺序创建。
- 初始测试数据能完整导入。
- 重复执行种子数据不会因主键冲突中断。

### 2. SQL 验收脚本

```bash
psql -v ON_ERROR_STOP=1 -d fcqa -f db/verify_basic_queries.sql
```

验证目标：

- 表数据量检查。
- 校区到建筑、建筑到设施的层级查询。
- 教师到课程的多表关联查询。
- 活动到设施、建筑、校区的关联查询。
- 建筑分类统计、热门查询类别统计、热门活动统计。

## 四、人工演示检查清单

课堂展示前建议按以下顺序检查：

1. 打开 `http://127.0.0.1:8000`，确认右上角健康状态显示正常。
2. 在“数据查询”中分别查看校区、建筑、设施、课程、活动。
3. 新增一个建筑，确认左侧列表刷新后可查询到。
4. 编辑刚新增的建筑，确认修改结果立即展示。
5. 新增一个设施并绑定到该建筑。
6. 新增一个活动并绑定到该设施。
7. 写入一条查询记录。
8. 点击“关键 SQL 效果”的执行按钮，确认每条 SQL 都展示查询文本和结果表格。
9. 删除演示中新增的活动、设施、建筑，确认数据可清理。

## 五、剩余风险与应对

- PostgreSQL 未启动或连接串错误时，前端会显示数据库未连接；演示前应先执行 `GET /api/health` 或查看页面右上角状态。
- 外键约束会阻止删除仍被引用的数据，这是预期行为。演示删除时建议删除课堂中新建的无依赖记录。
- 中文全文搜索仍以 `ILIKE` 和 trigram 模糊匹配为主，`tsvector` 生成列作为后续扩展保留。
- 演示模式不写入 PostgreSQL，仅用于无数据库环境下预览界面和 API 流程；最终评审建议使用 PostgreSQL 模式。
