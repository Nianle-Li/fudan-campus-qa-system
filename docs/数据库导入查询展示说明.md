# 数据库导入查询展示说明

本文档用于课堂或验收时快速展示数据库可以从空库初始化、导入测试数据，并支持基础查询、统计和写入联调。

## 环境与启动

数据库使用 PostgreSQL 16，Docker Compose 已配置在 `docker-compose.yml` 中。

```bash
docker compose up -d db
```

默认连接信息：

- 数据库：`fcqa`
- 用户：`postgres`
- 密码：`postgres`
- 本机端口：`5432`

首次创建数据卷时，Compose 会执行 `docker/postgres/01-init.sql`，进而调用 `db/init.sql`，完成建表、索引创建和初始 SQL 测试数据导入。

如果需要使用规模更大的 CSV 演示数据，可在数据库启动后执行 CSV 统一初始化入口。该入口会串联建表、清空演示表、导入 `db/seeds/csv/` 中的 CSV，并重置 `SERIAL` 序列：

```bash
docker compose exec -w /opt/fcqa db psql -P pager=off -v ON_ERROR_STOP=1 -U postgres -d fcqa -f db/init_csv.sql
```

## 查询展示点

`db/verify_basic_queries.sql` 可用于 SQL 初始数据和 CSV 演示数据，覆盖以下典型查询：

- 按校区查询建筑：例如查询“邯郸校区”下的教学楼、图书馆、食堂等建筑。
- 按建筑查询设施：例如查询“第二教学楼”中的教室和自习室开放时间。
- 按教师查询课程：例如查询“李芳”老师本学期教授的课程。
- 查询近期活动：展示活动、时间、校区、建筑和设施的多表关联结果。
- 统计建筑类型：按校区和建筑类型聚合统计。
- 统计查询日志：按查询类别统计用户常问内容。
- 统计热门活动：按参与记录统计活动热度。
- 事务写入联调：插入一条临时用户记录后回滚，用于证明数据库支持基础写入且不会污染演示数据。

```bash
docker compose exec db psql -P pager=off -v ON_ERROR_STOP=1 -U postgres -d fcqa -f /opt/fcqa/db/verify_basic_queries.sql
```

如果需要分步演示而不是一次性执行整份脚本，可进入交互式 `psql`：

```bash
docker compose exec db psql -U postgres -d fcqa
```

进入后，可手动执行 `db/verify_basic_queries.sql` 中的单条查询语句，按展示节奏逐条说明。展示结束后输入 `\q` 退出。

如果只想快速确认查询脚本是否运行成功，可以观察最后一条“热门活动”统计查询是否正常返回结果，且执行过程中没有出现 `ERROR`。

## 写入测试

可用事务内写入并回滚的方式展示基础存储能力，不污染演示数据：

```bash
docker compose exec db psql -P pager=off -v ON_ERROR_STOP=1 -U postgres -d fcqa -c "BEGIN; INSERT INTO users(user_id, name, department) VALUES (999999, '演示用户', '数据库测试') RETURNING user_id, name, department; SELECT count(*) AS user_count_in_tx FROM users; ROLLBACK;"
```

预期结果：插入语句返回一条临时用户记录，事务内用户数变为 9，随后 `ROLLBACK` 恢复原始数据。

## 索引与扩展检查

检查模糊检索扩展和索引是否已经创建：

```bash
docker compose exec db psql -P pager=off -v ON_ERROR_STOP=1 -U postgres -d fcqa -c "SELECT extname FROM pg_extension WHERE extname IN ('pg_trgm','unaccent') ORDER BY extname; SELECT count(*) AS fcqa_index_count FROM pg_indexes WHERE schemaname = 'public' AND indexname LIKE 'idx_%';"
```

已验证结果包含 `pg_trgm`、`unaccent`，并创建 27 个项目索引。

## 清理重建

如果需要从空库重新演示初始化过程：

```bash
docker compose down -v
docker compose up -d db
```
